# 快速创建弹窗设计规格

> 日期：2026-04-01
> 执行者：Codex

## 1. 背景

当前快速创建弹窗仍是“标题输入 + 类型 ChoiceChip + 系统下拉框”的旧结构。它能完成基础创建，但视觉质感粗糙，和当前任务页的羊皮纸 / 森林风格不统一；同时它无法在一个弹窗内完成“新建主线并同步添加支线”的层级创建。

## 2. 目标

- 保留同一快速创建弹窗，不新增平行入口。
- 在同一弹窗内支持三条路径：
  - 新建主线，并可同步添加 1..N 个支线任务。
  - 直接挂到已有主线创建支线任务。
  - 创建日常任务，并设置每日 HH:mm 截止时间。
- 统一弹窗、输入框、选择器、任务类型颜色，与当前 QuestTheme 背景风格一致。
- 替换当前生硬的系统下拉框表现，让“已有主线选择”也符合弹窗整体气质。

## 3. 非目标

- 不改动任务语义：Main_Quest / Side_Quest / Daily 的业务定义保持不变。
- 不新增新的控制器或数据库接口；继续复用现有 createManualQuest 能力。
- 不拆分出新的独立页面或二级弹窗流程。

## 4. 交互结构

### 4.1 模式分区

弹窗主体改为三张主模式卡：

1. 新建主线并添加支线
2. 挂到已有主线
3. 日常任务

用户点击卡片后，下面展示对应表单区域，而不是继续使用系统感很强的下拉组件。

### 4.2 新建主线并添加支线

- 主线标题必填。
- 下方内嵌支线草稿列表。
- 支持：
  - 只创建主线
  - 创建主线 + 多个支线
- 支线输入为轻量卡片列表，保留“添加一条支线”“删除该条支线”。

### 4.3 挂到已有主线

- 保留现有能力，不能删除。
- 在同一弹窗内完成。
- 使用自定义主线选择卡片或列表，而不是 DropdownButtonFormField。
- 若当前无可挂载主线，需要明确提示“请先创建主线”。

### 4.4 日常任务

- 保留单任务标题输入。
- 截止时间仍为每日 HH:mm，而不是日期。
- 继续使用 showTimePicker，但在弹窗内以更统一的时间信息卡展示当前状态与清除操作。

## 5. 视觉方向

- 综合色板：暖白 / 羊皮纸 / 森林绿 / 柔和金色。
- 主线、支线、日常任务颜色要更柔和、低饱和、与 QuestTheme 协调：
  - 主线：偏金色 / 琥珀暖调
  - 支线：偏森林绿 / 蓝绿过渡
  - 日常：偏晨光青蓝
- 组件风格：
  - 大圆角
  - 轻描边
  - 弱阴影
  - 低对比渐变
  - 输入区与说明区分层清晰

## 6. 技术方案

- 主改动文件：`frontend/lib/features/quest/screens/home_page.dart`
- 复用：
  - `QuestDialogShell`
  - `QuestDialogInfoCard`
  - `QuestDialogPrimaryButton`
  - `QuestTheme` / `AppColors` / `AppTextStyles`
- 提交策略：
  - 主线模式：先创建 Main_Quest，再用返回的主线 id 顺序创建支线。
  - 挂载模式：直接创建 Side_Quest，并带上已选主线 id。
  - 日常模式：直接创建 Daily，并带上 dailyDueMinutes。

## 8. 验收标准

- 快速创建弹窗中明确存在三条创建路径。
- 用户可以在同一弹窗内创建“新主线 + 多个支线”。
- 用户仍可在同一弹窗内把支线直接挂到已有主线。
- 日常任务仍可设置每日 HH:mm 截止。
- 现有系统下拉框不再是支线挂载的主要 UI。
- 相关格式化、分析和测试通过，并有留痕文档。
