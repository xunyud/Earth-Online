# Earth Online

Earth Online is a memory-aware productivity game that turns everyday planning into a living quest log. It blends a Flutter client, Supabase Edge Functions, and a lightweight backend so users can capture real tasks, revisit recent context, and receive guidance shaped by past behavior and memory.

> Think of it as a quest board and an assistant that remembers, supports, and keeps you company.

## Core Experience

### 1. A Quest Board For Real Life

Earth Online turns ordinary planning into a game-like quest flow:

- create, organize, and complete quests on a structured board
- accumulate XP, levels, rewards, inventory items, and achievement progress
- keep work grounded in real tasks instead of abstract AI chat alone

### 2. A Memory-Aware Assistant

This assistant is not a stateless chatbot. It reads recent signals and uses them to shape responses:

- recent memory and behavior signals are packed into guide prompts
- the assistant can suggest recovery actions or forward motion based on context and memory
- daily events, proactive messages, and reflection flows all reuse the same memory-aware layer

### 3. WeChat As A Real Interaction Surface

The current WeChat flow is designed around recording tasks whenever they happen:

- bind a WeChat account from the app
- send normal text to continue through the existing task-capture path
- ask for help in WeChat and receive an assistant-style reply
- accept the latest suggestion directly into the quest tree

### 4. A Persistent Personal Loop

Beyond chat and task capture, the product already includes:

- life diary and recycle bin flows for preserving short-term context
- profile customization with editable guide name and avatar
- stats, a reward shop, and inventory systems that make progress visible
- user portraits generated by the assistant from long-term memory

## Repository Guide

### Main directories

- `frontend/`: Flutter application, including the quest board, guide panel, WeChat binding UI, profile, diary, rewards, stats, and achievements
- `backend/`: Node.js / Express service for webhook ingestion and debounced processing
- `supabase/`: migrations, shared guide logic, and Edge Functions for task parsing, guide chat, memory sync, weekly summaries, portrait generation, and WeChat flows
- `docs/`: product notes, implementation plans, and supporting references

## Tech Stack

- Flutter + Dart
- Supabase
- Node.js + Express
- Redis
- TypeScript
